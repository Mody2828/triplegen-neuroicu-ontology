CORE CONSTRAINTS

Source fidelity
Extract only information explicitly stated in the current text.
Do not infer, guess, paraphrase into new concepts, or use outside medical knowledge.
If support is unclear, omit the item.

Evidence anchor
Every extracted class, relation, and hierarchy edge must include an evidence field.
Evidence must be an exact verbatim phrase or sentence fragment from the text.

Ignore non-clinical and publication content
Do not extract classes, relations, or hierarchy from:
authors, affiliations, institutions, investigator lists
journal names, page numbers, DOI, ISBN, PMID, publication dates, copyright notices, URLs
organisational and governance content such as meetings, steering groups, membership, centres, administration, ethics approval, or funding
database or infrastructure content such as forms, software, websites, transfer workflows, anonymisation workflows, validation workflows, or coordinating-centre logistics
general research-process wording such as study design, feasibility exercise, trial setup, or data-collection procedure
If a sentence or paragraph is mainly about these topics, ignore it unless it contains a clearly named allowed clinical concept.

Priority rule
Classes are the priority.
Extract relations only when the wording clearly supports the relation and both endpoints.
If relation support is weak, return no relations.
Extract hierarchy only when there is an explicit lexical cue.

Confidence rule
If an allowed clinical concept is clearly named in the text, include it even if mentioned briefly.
Omit only genuinely ambiguous, weakly supported, or non-clinical items.

Hierarchy rule
Extract hierarchy only when the text contains explicit lexical cues such as:
is a
type of
kind of
such as
include:
includes:
Do not infer hierarchy from loose narrative wording, thresholds, examples without lexical cues, or procedural descriptions.

Anti-anchoring rule
Use the retrieved example below as format and behaviour guidance only.
Do not copy labels, relations, or hierarchy from the example unless the current text explicitly supports them.

Conservative extraction
Prefer under-extraction to hallucination.
When uncertain, omit.

4 Discussion and Future Work
We have shown that our approach can be successfully applied to assess medical
data and illustrates the benefit of additional contextual information provided by
linking data. Using a simple approach, a large number of acceptable range errors
are identified; however, when the context of the data is considered, many are reclassified or new errors are identified. In the future we plan to investigate: further

a patient but not recorded; and using High Performance Computing to improve
performance times, e.g. it took 2 hours 8 mins to apply the Medical Disorders
rules which clearly would not be practical if used regularly in a clinical setting.
